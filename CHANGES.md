# Changelog

All notable changes to this project will be documented in this file.

## [Upcoming Version]
### Added

### Changed
* **匯入備份的驗證更嚴格**：還原備份時除了結構外，也會檢查名稱、備註、標籤與待購備註的內容型別，避免損壞的備份檔寫入後導致擴充功能在所有支援網站上靜默失效。

### Fixed
* **中文輸入法 Enter 選字誤觸修正**：使用注音等輸入法輸入標籤或名單時，組字中按 Enter 選字不再把未完成的組字內容誤加為標籤或條目。
* **儲存失敗不再靜默**：待購備註／標籤儲存失敗（例如超過同步空間單一項目 8KB 上限）時，會在頁面顯示錯誤提示並保留編輯中的內容，不再顯示「已儲存」假象；管理頁的新增、編輯、刪除與標籤操作失敗時也會跳出錯誤訊息。
* **待購清單自動清理防誤刪**：自動清理已移除書籍的備註／標籤前，改為需連續兩輪確認頁面書單一致才執行，並在偵測不到任何書籍 ID（如站方改版）時完全跳過，避免頁面未完整載入時誤刪資料。
* **「自動關閉試讀提醒」關閉後即時生效**：在設定頁關閉此選項後，閱讀器的試讀計時器會立即恢復站方原有行為，不再需要重新載入頁面。
* **設定頁版號顯示修正**：設定頁頁尾版號長期卡在 v0.2.0 未跟隨實際版本更新，現已修正並同步至目前版本。

### Changed

### Fixed
* **待購清單備註／標籤未顯示修正**：在書籍詳細頁加入待購清單並儲存備註與標籤後，切換至待購清單頁面時資料未正確顯示。此問題在較慢的裝置或跨裝置同步時特別容易發生。

## [0.5.0] - 2026-06-08
### Added
* **待購清單最優價格提示**：Readmoo 待購清單頁每本書的價格下方，現在會顯示最優購買法提示（輕量綠色標籤格式）。停止銷售的書籍不顯示提示。
* **待購清單 Tag 篩選欄位**：待購清單頁上方新增標籤篩選狀態條，一進頁面即列出所有使用中的標籤。點擊標籤可啟用／停用篩選（支援多選，AND 交集邏輯）；無符合結果時顯示提示訊息；點擊「清除標籤篩選」一鍵恢復顯示全部書籍。切換分頁後篩選狀態自動重置。
* **待購清單標籤自動保留**：書籍購買後從待購清單移除時，其標籤若已無其他書籍使用，將自動儲存為常用標籤（不隨書籍消失）。常用標籤可在管理介面「待購清單標籤」頁面重命名或刪除，並顯示為 `(0 本)`。標籤被新增至任一書籍後自動從常用標籤移除。

## [0.4.0] - 2026-06-04
### Added
* **自動關閉試讀提醒**（讀墨閱讀器）：在設定頁「網站設定 → 讀墨」開啟後，試讀時的「喜歡這本書嗎？加入購物車吧！」及「該讓眼睛休息一下囉！」對話框將自動被阻擋，不再打斷閱讀。
* **跨裝置資料同步**：改用 `chrome.storage.sync` 儲存所有資料，同一 Chrome 帳號登入的多台裝置可自動同步黑白名單、待購清單備註與標籤。更新後首次開啟頁面時，現有資料會自動從舊儲存空間搬移，無需手動操作。
* **最優價格擴展至列表與活動頁**：最優購買法試算現在支援更多頁面類型。列表卡片（每日特惠書等）以輕量綠色標籤顯示最佳選項；讀墨活動頁（`/campaign/*`）的大張宣傳卡片顯示完整折扣下拉選單，延伸閱讀區塊則顯示輕量標籤。


## [0.3.0] - 2026-05-26

## [0.2.0] - 2026-05-16
### Added
* **博客來 (books.com.tw) 名單支援**：黑白名單功能擴展至博客來，在書籍詳情頁（`.type02_p01_wrap`）對作者與出版社套用 `teh-blacklisted-text` / `teh-whitelisted-text` 樣式，並將書名標記為 `teh-blacklisted-title`；列表頁亦支援名單標記（受限於博客來頁面結構，列表頁僅有可連結的作者/出版社資訊才能被標記）。
* **Kobo TW 名單支援**：黑白名單功能擴展至 Kobo 台灣站，支援書籍詳情頁（作者標記 + 書名打叉）與搜尋頁（作者標記）。Kobo 不顯示出版社連結，故出版社名單不適用。
* **領書額度適用性自動偵測**：自動識別書籍是否支援「領書額度」兌換（如排除不適用的套書），提高價格試算的準確性。
* **Tag 標籤功能**：黑名單、優良名單的出版社與作者/譯者條目，以及待購清單書籍，現在均可加上自訂標籤。標籤以 chip 形式顯示，輸入時提供 autocomplete 建議。管理中心新增「標籤管理」頁，可對名單標籤與待購清單標籤分別進行重命名、刪除（批次套用至所有含該標籤的條目）。
* **名單條目行內編輯**：管理中心的名單條目支援直接點擊鉛筆按鈕進入行內編輯模式，可修改名稱、備註與標籤，無需刪除後重新新增。
* **待購清單 tag 支援**（Readmoo）：在書籍詳情頁將書籍加入待購清單時，可同時輸入備註與標籤；待購清單頁面也可點擊各書籍的備註區塊，行內編輯備註與標籤。

### Changed
* **資料結構升版（0.2.0）**：名單條目新增 `tags` 欄位；待購清單新增獨立儲存鍵 `wishlistTags`；匯出備份包含 `schemaVersion` 以支援未來版本遷移。舊有備份檔匯入時會自動套用遷移。
* **價格抓取邏輯優化**：改採更穩定的「容器 + 文字過濾」策略，提升 Readmoo 改版時的耐受度。

### Fixed
* **待購清單備註遺失**：修正從書籍詳情頁的備註輸入框儲存後，在待購清單頁面卻看不到備註的問題。根本原因為書籍 ID 的取得方式不一致：書籍頁優先使用 `data-readmoo-id` 屬性（格式未必與 URL 相同），而待購清單頁則從連結 URL 取得，導致兩者 key 不吻合。現已統一改由 URL 取得 ID。
* **讀墨確認視窗關閉備註輸入框**：修正點擊「知道了」或關閉讀墨的「成功加入待購清單」對話框時，會連帶關閉小幫手備註輸入框的問題。利用 capture phase 先於讀墨 handler 執行的特性，偵測到 `[role="dialog"]` 內的點擊時略過關閉動作。
* **書籍頁移除待購清單不再清除備註**：修正在書籍詳情頁點擊「移除待購清單」時，會一併刪除已儲存備註與標籤的問題。資料現在會保留，下次重新加入時仍可看到先前輸入的內容。


## [0.1.0] - Initial Release
### Added
* **價格試算小幫手**：在 Readmoo 讀墨書籍詳情頁自動計算最划算的購買方式（75折、扣50元、或領書額度換算）。
* **黑白名單管理**：可將作者或出版社加入黑/白名單，自動在網頁上打叉或變淡隱藏，淨化購書畫面。
* **待購清單備註**：可在購物車或待購清單中，為想買的書加入自訂備註（如：為什麼想買、推薦人等）。
* **資料備份與還原**：管理介面支援將黑白名單與備註資料匯出為 JSON 格式，並支援拖曳匯入還原。
